# Policies

- Last updated: 2026-04-11

Policies define normative rules that contributors and tooling must follow.

## When to write one

Write a policy for contributor standards, workflow rules, and requirements Tau may enforce in
tooling or CI.

Do not write a policy for architecture decisions, step-by-step tutorials, or temporary notes.

## Naming

- Short lowercase kebab-case: `repository-workflows.md`.
- One policy per file when practical.

## Required format

Every policy follows the [template](./TEMPLATE.md).

Required sections: Status, Last updated, Applies to, Rules, Exceptions, Enforcement.

Optional sections: Examples, See also.
